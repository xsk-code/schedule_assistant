## 1. 产品概述

### 1.1 产品定位

**四化节奏师**是一款基于紫微斗数四化能量的AI任务拆分助手。它通过分析用户输入的任务描述，结合当日的禄权科忌四星能量，智能地为用户提供任务的最小行动路径和个性化行动建议。

与市面上普通的任务管理工具不同，**四化节奏师**的核心特色在于：

- ✨ **能量导向**：结合传统紫微斗数的四化能量分析
- 🎯 **智能拆分**：AI驱动的任务最小化分解
- 💡 **个性化建议**：根据当日能量给出最优行动策略
- 🌈 **四维分析**：从禄（机遇）、权（执行）、科（计划）、忌（风险）四个维度全方位解析任务

### 1.2 产品愿景

让每个人都能掌握自己的时间节奏，在正确的时间做正确的事情。通过传统智慧与现代AI的结合，帮助用户：

1. **化繁为简**：将复杂任务拆解为可执行的最小行动
2. **顺势而为**：根据当日能量选择最优的任务切入点
3. **趋吉避凶**：提前识别任务中的风险点（忌）并做好准备
4. **事半功倍**：在能量最高的维度投入最大精力

### 1.3 核心价值

| 维度 | 价值说明 |
|------|----------|
| **时间价值** | 快速定位任务的最佳切入点，减少决策时间 |
| **心理价值** | 降低任务复杂度带来的焦虑感，提升行动力 |
| **决策价值** | 提供四维分析视角，帮助用户做出更明智的行动决策 |
| **文化价值** | 将传统紫微斗数智慧与现代生产力工具结合，焕发新活力 |

## 2. 用户画像与使用场景

### 2.1 用户画像

#### 目标用户群体

1. **职场人士**
   - 特点：工作任务繁杂，需要高效的任务管理方法
   - 痛点：面对复杂项目不知从何下手，经常陷入决策瘫痪
   - 需求：快速将大任务拆解为可执行的步骤

2. **创业者/自由职业者**
   - 特点：需要自主管理时间和任务，缺乏外部约束
   - 痛点：容易拖延，任务优先级混乱
   - 需求：个性化的任务建议，提升自我管理能力

3. **对传统文化感兴趣的人群**
   - 特点：相信命理或传统文化，希望将其应用于生活
   - 痛点：传统命理知识晦涩难懂，难以实际应用
   - 需求：将四化能量以通俗易懂的方式指导日常行动

4. **学生群体**
   - 特点：学习任务繁重，需要合理规划时间
   - 痛点：面对多个科目和作业，不知如何分配精力
   - 需求：将复杂的学习计划拆解为具体的行动步骤

#### 用户核心需求排序

| 优先级 | 需求 | 描述 |
|--------|------|------|
| P0 | 任务拆解 | 将复杂任务拆解为可执行的最小步骤 |
| P0 | 能量结合 | 根据当日四化给出行动建议 |
| P1 | 历史记录 | 查看和管理历史任务分析 |
| P1 | 多任务管理 | 支持同时分析多个任务 |
| P2 | 任务导出 | 将分析结果导出为图片或文本 |
| P2 | 分享功能 | 分享任务分析结果给他人 |

### 2.2 使用场景

#### 场景一：职场项目启动

**用户**：产品经理小张
**时间**：周一早晨
**场景**：
> 小张刚接到一个新的产品项目，需要从0到1规划整个产品 roadmap。面对这个复杂的任务，他打开了"四化节奏师"，输入："规划新产品从需求调研到上线的完整 roadmap"。

**系统输出**：
- 📅 **今日四化**：廉贞化禄、破军化权、武曲化科、太阳化忌
- 🎯 **最佳切入点**：今天禄在廉贞（社交/人脉），适合从用户调研和需求访谈开始
- ⚠️ **注意事项**：忌在太阳（表现/公开），今天不适合做公开的方案汇报
- 📋 **最小行动路径**：
  1. 列出需要访谈的用户名单（禄：社交机会）
  2. 预约3位核心用户的访谈时间（权：执行决断）
  3. 整理访谈提纲（科：计划准备）
  4. 准备录音设备，确保访谈顺利（忌：避免疏漏）

#### 场景二：学生备考规划

**用户**：大学生小李
**时间**：考试周前两周
**场景**：
> 小李需要同时准备3门课程的期末考试，感觉无从下手。他输入："准备高等数学、线性代数、概率论三门课程的期末考试复习"。

**系统输出**：
- 📅 **今日四化**：太阴化禄、天同化权、天机化科、巨门化忌
- 🎯 **最佳切入点**：禄在太阴（内在/思考），适合从需要深度理解的高数开始
- ⚠️ **注意事项**：忌在巨门（口舌/争论），今天避免和同学讨论难题，容易产生误解
- 📋 **复习计划**：
  1. 高数：重点复习极限和导数章节（禄：深度思考）
  2. 线代：整理矩阵运算的常见题型（权：执行练习）
  3. 概率：整理公式清单（科：系统化）
  4. 避免：今天不要组学习小组讨论，容易越讨论越混乱（忌）

#### 场景三：创业者日常任务

**用户**：创业者小王
**时间**：每个工作日早晨
**场景**：
> 小王习惯每天早晨用5分钟规划一天的工作。他输入自己的待办事项清单："今天需要完成产品原型、回复投资人邮件、面试候选人和团队周会"。

**系统输出**：
- 📅 **今日四化**：贪狼化禄、太阴化权、右弼化科、天机化忌
- 🎯 **任务优先级**：
  1. ✅ **面试候选人**（禄：贪狼主人才/机遇，今天面试容易发现优秀人才）
  2. ✅ **产品原型**（权：太阴主执行，今天执行力强）
  3. ✅ **团队周会**（科：右弼主协作，今天开会沟通顺畅）
  4. ⏳ **回复投资人邮件**（忌：天机主变动，今天写邮件容易表达不清，建议明天再写）
- ⚠️ **今日提醒**：忌在天机，今天避免做需要精细逻辑的文字工作，容易出错

### 3.2 核心功能模块

#### 3.2.1 任务输入模块

**功能描述**：提供用户输入任务描述的界面

**功能详情**：

| 功能项 | 描述 |
|--------|------|
| **输入方式** | 多行文本输入框，支持粘贴 |
| **输入提示** | 占位符提示："请输入您想要分析的任务，例如：'准备下周的产品发布会'" |
| **输入限制** | 最少5字，最多500字 |
| **示例建议** | 提供3个输入示例供用户参考点击 |
| **历史快捷输入** | 可快速选择最近3条历史任务 |

**输入示例**：

1. "规划新产品从需求调研到上线的完整 roadmap"
2. "准备高等数学、线性代数、概率论三门课程的期末考试复习"
3. "今天需要完成产品原型、回复投资人邮件、面试候选人和团队周会"

**交互流程**：

```
用户输入文本 → 实时校验字数 → 点击"开始分析"按钮 → 显示加载动画
```

#### 3.2.2 四化计算模块

**功能描述**：根据当前日期计算当日的天干和四化星

**技术基础**：
- 使用现有的 `lunar.js` 库进行公历到农历的转换
- 使用十天干四化规则表进行映射

**十天干四化规则表**：

| 天干 | 化禄 | 化权 | 化科 | 化忌 |
|------|------|------|------|------|
| 甲 | 廉贞 | 破军 | 武曲 | 太阳 |
| 乙 | 天机 | 天梁 | 紫微 | 太阴 |
| 丙 | 天同 | 天机 | 文昌 | 廉贞 |
| 丁 | 太阴 | 天同 | 天机 | 巨门 |
| 戊 | 贪狼 | 太阴 | 右弼 | 天机 |
| 己 | 武曲 | 贪狼 | 天梁 | 文曲 |
| 庚 | 太阳 | 武曲 | 太阴 | 天同 |
| 辛 | 巨门 | 太阳 | 文曲 | 文昌 |
| 壬 | 天梁 | 紫微 | 左辅 | 武曲 |
| 癸 | 破军 | 巨门 | 太阴 | 贪狼 |

**四化能量含义**：

| 四化 | 含义 | 在任务中的应用 |
|------|------|----------------|
| **禄** | 机遇、资源、收获 | 适合开拓、社交、获取资源的行动 |
| **权** | 权力、执行、决断 | 适合决策、执行、推动进展的行动 |
| **科** | 名声、学习、计划 | 适合规划、学习、系统化的行动 |
| **忌** | 阻碍、风险、注意 | 需要谨慎、避免、预防问题的领域 |

**计算流程**：

```
输入：公历日期（YYYY-MM-DD）
↓
1. 使用 lunar.js 转换为农历日期
↓
2. 获取农历日期的干支纪日
↓
3. 提取天干（干支的第一个字）
↓
4. 根据天干查找四化规则表
↓
输出：{
  date: "2026-04-17",
  lunar_date: "农历三月初一",
  day_ganzhi: "甲子日",
  day_gan: "甲",
  day_zhi: "子",
  sihua: {
    lu: "廉贞",
    quan: "破军",
    ke: "武曲",
    ji: "太阳"
  }
}
```

#### 3.2.3 AI分析模块

**功能描述**：调用硅基流动API，结合四化能量进行任务分析和拆分

**核心能力**：

1. **任务理解**：理解用户输入的任务描述
2. **能量融合**：将当日四化能量融入分析逻辑
3. **智能拆分**：将任务拆解为最小可执行步骤
4. **四维分析**：从禄权科忌四个维度给出建议

**AI Prompt 设计思路**：

```
【系统角色】
```

#### 5.4.1 四化计算服务 (sihuaService.ts)

```typescript
import { Solar } from '../../vendor/lunar.js';
import { SIHUA_RULES } from '../constants/sihuaRules';

export interface SihuaInfo {
  solarDate: string;
  lunarDate: string;
  dayGanZhi: string;
  dayGan: string;
  dayZhi: string;
  sihua: {
    lu: string;
    quan: string;
    ke: string;
    ji: string;
  };
}

/**
 * 根据公历日期计算四化信息
 */
export function calculateSihua(year: number, month: number, day: number): SihuaInfo {
  const solar = Solar.fromYmd(year, month, day);
  const lunar = solar.getLunar();
  
  // 获取干支纪日
  const dayGanZhi = lunar.getDayInGanZhiExact2 
    ? lunar.getDayInGanZhiExact2() 
    : lunar.getDayInGanZhiExact();
  
  const dayGan = dayGanZhi.charAt(0);
  const dayZhi = dayGanZhi.substring(1);
  
  // 查找四化规则
  const sihua = SIHUA_RULES[dayGan];
  
  if (!sihua) {
    throw new Error(`未找到天干 ${dayGan} 对应的四化规则`);
  }
  
  return {
    solarDate: `${year}-${String(month).padStart(2, '0')}-${String(day).padStart(2, '0')}`,
    lunarDate: lunar.toString(),
    dayGanZhi,
    dayGan,
    dayZhi,
    sihua: {
      lu: sihua.lu,
      quan: sihua.quan,
      ke: sihua.ke,
      ji: sihua.ji,
    },
  };
}

/**
 * 获取今日四化信息
 */
export function getTodaySihua(): SihuaInfo {
  const now = new Date();
  return calculateSihua(now.getFullYear(), now.getMonth() + 1, now.getDate());
}
```

```typescript
/**
 * 调用AI进行任务分析
 */
export async function analyzeTask(
  task: string,
  sihuaInfo: SihuaInfo,
  apiKey: string
): Promise<AnalysisResult> {
  const prompt = buildAnalysisPrompt(task, sihuaInfo);
  
  const response = await fetch('https://api.siliconflow.cn/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${apiKey}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: [
        {
          role: 'user',
          content: prompt,
        },
      ],
      response_format: { type: 'json_object' },
      temperature: 0.7,
      max_tokens: 4000,
    }),
  });
  
  if (!response.ok) {
    const error = await response.json().catch(() => ({}));
    throw new Error(error.error?.message || `API调用失败: ${response.status}`);
  }
  
  const data = await response.json();
  const content = data.choices?.[0]?.message?.content;
  
  if (!content) {
    throw new Error('API返回内容为空');
  }
  
  try {
    return JSON.parse(content) as AnalysisResult;
  } catch {
    throw new Error('解析AI响应失败');
  }
}
```

| **API Key 安全问题** | 低 | 高 | 1. 仅本地存储，不上传<br>2. 提供清除功能<br>3. 明确告知用户 |
| **lunar.js 兼容性问题** | 低 | 中 | 1. 充分测试日期转换<br>2. 准备备选方案 |
| **提示词效果不佳** | 中 | 高 | 1. 多轮迭代优化<br>2. 准备多套备选提示词<br>3. 收集用户反馈持续优化 |

### 8.2 产品风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| **用户对四化理解不足** | 高 | 中 | 1. 提供详细的说明文档<br>2. 结果中加入简要解释<br>3. FAQ 解答常见问题 |
| **分析结果不符合预期** | 中 | 高 | 1. 持续优化提示词<br>2. 支持重新分析<br>3. 收集用户反馈 |
| **API 成本问题** | 低 | 中 | 1. 用户自行配置 API Key<br>2. 提示用户关注用量<br>3. 提供成本估算 |

### 8.3 应对策略总结

1. **防御性设计**：所有外部依赖都要有错误处理和降级方案
2. **用户教育**：产品内提供足够的说明和帮助
3. **快速迭代**：MVP 后快速收集反馈，持续优化
4. **数据安全**：用户数据仅保存在本地，保护隐私
