## 一、项目概述

将现有 Web 版「四化节奏师」迁移为微信小程序，保持核心功能不变，UI 采用 **Claude 官网极简风格**。

### 1.1 核心目标

- 功能对齐：保持 Web 版全部功能（简单模式、思考模式、历史记录、设置）
- 风格重塑：Claude 极简风格，大量留白，信息层级清晰
- 开发优先：当前为开发阶段，小程序直连 SiliconFlow API，后续上线时再接入后端代理

### 1.2 与 Web 版的关键差异

| 维度 | Web 版 | 小程序版 |
|------|--------|---------|
| **API Key** | 用户自行配置，存 localStorage | 内置配置，用户无需关心 |
| **数据存储** | localStorage（最多50条） | wx.setStorageSync（本地存储） |
| **四化计算** | 前端 lunar.js | 前端 lunar-javascript（同） |
| **AI 请求** | 前端直连 SiliconFlow | 前端直连 SiliconFlow（开发阶段） |
| **模型选择** | 6个模型可选 | 默认配置，无需选择 |
| **分享** | 复制 JSON | 小程序分享卡片 + 生成图片 |

> 💡 开发阶段小程序直连 SiliconFlow API（需在微信开发者工具中勾选「不校验合法域名」），上线前再接入后端代理。

## 二、技术选型

| 层级 | 技术方案 | 理由 |
|------|---------|------|
| **小程序框架** | **Taro 4 + React + TypeScript** | 现有项目基于 React，Taro 可最大程度复用组件逻辑和 hooks |
| **样式方案** | **Tailwind CSS (taro-plugin-tailwind)** | 与现有项目一致，Claude 极简风格用 utility-first 最高效 |
| **状态管理** | **Zustand** | 轻量、TypeScript 友好，替代现有 useLocalStorage |
| **AI API** | **前端直连 SiliconFlow** | 开发阶段直接调用，上线前迁移至后端代理 |
| **农历计算** | **lunar-javascript** | 现有 vendor/lunar.js 的 npm 包版本 |

## 四、数据与网络设计

### 4.1 API 调用方式

开发阶段小程序直接调用 SiliconFlow API，与现有 Web 版逻辑一致：

| 功能 | 调用方式 | 说明 |
|------|---------|------|
| 任务分析 | 前端 → SiliconFlow `/chat/completions` | 使用 `promptBuilder.ts` 构建 prompt |
| 思考模式追问 | 前端 → SiliconFlow `/chat/completions` | 使用 `conversationService.ts` 逻辑 |
| 四化计算 | 前端本地计算 | `lunar-javascript` + `sihuaRules.ts` |
| 历史记录 | `wx.setStorageSync` | 本地存储，最多50条 |

### 4.2 API Key 管理

开发阶段 API Key 内置在代码配置中（仅用于开发），不暴露给用户：

```typescript
// constants/appConfig.ts
export const APP_CONFIG = {
  API_BASE_URL: 'https://api.siliconflow.cn/v1',
  API_KEY: process.env.TARO_APP_API_KEY,
  DEFAULT_MODEL: 'deepseek-ai/DeepSeek-V3.2',
  MAX_TOKENS: 4000,
  TEMPERATURE: 0.7,
  API_TIMEOUT: 60000,
  MAX_HISTORY_COUNT: 50,
  MIN_TASK_LENGTH: 5,
  MAX_TASK_LENGTH: 500,
};
```

> ⚠️ API Key 通过 Taro 环境变量 `TARO_APP_API_KEY` 注入，不硬编码在代码中。上线前迁移至后端代理。

### 4.3 本地存储设计

| 存储键 | 类型 | 说明 |
|--------|------|------|
| `sihua_history` | `HistoryRecord[]` | 历史记录，最多50条 |
| `sihua_settings` | `AppConfig` | 用户设置 |

### 4.4 开发环境配置

微信开发者工具需勾选以下选项：

```
右上角"详情" → 本地设置 → ☑️ 不校验合法域名、web-view、TLS版本及HTTPS证书
```

## 六、UI 设计规范（Claude 极简风格）

### 6.1 设计理念

参考 Claude 官网 (claude.ai) 的设计语言：

- **大量留白**：内容区域呼吸感强，不拥挤
- **极简配色**：以暖灰（stone）为基调，极少使用彩色
- **信息层级**：通过字重和灰度区分，而非颜色和边框
- **柔和圆角**：统一 16rpx 圆角，温和不生硬
- **微交互**：状态变化有细腻过渡，不突兀

### 6.2 配色系统

```
主色调（Claude 风格暖灰系）：
  --color-bg:          #FAFAF9     页面背景（暖白）
  --color-surface:     #FFFFFF     卡片背景
  --color-border:      #E7E5E4    分割线/边框
  --color-text-1:      #1C1917    主文字（暖黑）
  --color-text-2:      #57534E    次文字
  --color-text-3:      #A8A29E    辅助文字
  --color-text-4:      #D6D3D1    占位文字

四化能量色（低饱和度，融入整体）：
  --color-lu:          #4A7C59    禄（低饱和绿）
  --color-quan:        #92703A    权（低饱和琥珀）
  --color-ke:          #4A6B8A    科（低饱和蓝）
  --color-ji:          #9B4A4A    忌（低饱和红）

交互色：
  --color-primary:     #1C1917    主按钮（暖黑）
  --color-primary-bg:  #F5F5F4    主按钮 hover
```

### 6.3 字体规范

```
标题（Display）：  font-weight: 600  letter-spacing: -0.02em
  · 页面标题：    36rpx
  · 卡片标题：    32rpx
  · 区块标题：    28rpx

正文（Body）：     font-weight: 400
  · 主要内容：    28rpx  line-height: 1.6
  · 辅助说明：    24rpx  line-height: 1.5
  · 标签/徽章：   22rpx  font-weight: 500
```

### 6.4 间距系统

```
基于 8rpx 网格：
  xs:  8rpx    组件内微间距
  sm:  16rpx   组件内间距
  md:  24rpx   组件间间距
  lg:  32rpx   区块间间距
  xl:  48rpx   页面级间距
  2xl: 64rpx   大区块间距
```

## 八、关键代码迁移策略

| 现有模块 | 迁移方式 | 说明 |
|---------|---------|------|
| `sihuaRules.ts` / `starMeanings.ts` | **直接复用** | 纯数据常量，无平台依赖 |
| `promptBuilder.ts` | **直接复用** | 前端构建 prompt，与 Web 版一致 |
| `conversationService.ts` | **直接复用** | 前端直连 SiliconFlow，逻辑不变 |
| `aiService.ts` | **直接复用** | 前端直连 SiliconFlow，`fetch` → `Taro.request` |
| `sihuaService.ts` | **直接复用** | 前端本地计算，`lunar.js` → `lunar-javascript` |
| `useSihua.ts` | **Taro 适配** | 逻辑复用，React hooks 在 Taro 中可用 |
| `useAIAnalysis.ts` | **Taro 适配** | 逻辑复用，`fetch` 替换为 `Taro.request` |
| `useConversation.ts` | **Taro 适配** | 对话状态管理逻辑复用 |
| `types/index.ts` | **直接复用** | 类型定义与平台无关 |
| `ResultDisplay.tsx` | **Taro 适配重写** | JSX 结构可复用，样式改写 |
| `TaskInput.tsx` | **Taro 适配重写** | 交互逻辑复用，UI 简化 |

## 十、开发阶段规划

| 阶段 | 内容 | 产出 |
