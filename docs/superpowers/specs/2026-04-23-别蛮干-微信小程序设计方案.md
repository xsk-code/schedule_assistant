## 一、项目概述

将现有 Web 版「别蛮干」迁移为微信小程序，保持核心功能不变，UI 采用 **Claude 官网极简风格**。

### 1.1 核心目标

- 功能对齐：保持 Web 版全部功能（简单模式、思考模式、历史记录、设置）
- 风格重塑：Claude 极简风格，大量留白，信息层级清晰
- 开发优先：当前为开发阶段，小程序直连 SiliconFlow API，后续上线时再接入后端代理

### 1.2 与 Web 版的关键差异

| 维度 | Web 版 | 小程序版 |
|------|--------|---------|
| **API Key** | 用户自行配置，存 localStorage | 内置配置，用户无需关心 |
| **数据存储** | localStorage（最多50条） | wx.setStorageSync（本地存储） |
| **四化计算** | 前端 lunar.js | 前端 lunar-javascript（同） |
| **AI 请求** | 前端直连 SiliconFlow | 前端直连 SiliconFlow（开发阶段） |
| **模型选择** | 6个模型可选 | 默认配置，无需选择 |
| **分享** | 复制 JSON | 小程序分享卡片 + 生成图片 |

> 💡 开发阶段小程序直连 SiliconFlow API（需在微信开发者工具中勾选「不校验合法域名」），上线前再接入后端代理。

## 二、技术选型

| 层级 | 技术方案 | 理由 |
|------|---------|------|
| **小程序框架** | **Taro 4 + React + TypeScript** | 现有项目基于 React，Taro 可最大程度复用组件逻辑和 hooks |
| **样式方案** | **Tailwind CSS (taro-plugin-tailwind)** | 与现有项目一致，Claude 极简风格用 utility-first 最高效 |
| **状态管理** | **Zustand** | 轻量、TypeScript 友好，替代现有 useLocalStorage |
| **AI API** | **前端直连 SiliconFlow** | 开发阶段直接调用，上线前迁移至后端代理 |
| **农历计算** | **lunar-javascript** | 现有 vendor/lunar.js 的 npm 包版本 |

## 四、数据与网络设计

### 4.1 API 调用方式

开发阶段小程序直接调用 SiliconFlow API，与现有 Web 版逻辑一致：

| 功能 | 调用方式 | 说明 |
|------|---------|------|
| 任务分析 | 前端 → SiliconFlow `/chat/completions` | 使用 `promptBuilder.ts` 构建 prompt |
| 思考模式追问 | 前端 → SiliconFlow `/chat/completions` | 使用 `conversationService.ts` 逻辑 |
| 四化计算 | 前端本地计算 | `lunar-javascript` + `sihuaRules.ts` |
| 历史记录 | `wx.setStorageSync` | 本地存储，最多50条 |

### 4.2 API Key 管理

开发阶段 API Key 内置在代码配置中（仅用于开发），不暴露给用户：

```typescript
// constants/appConfig.ts
export const APP_CONFIG = {
  API_BASE_URL: 'https://api.siliconflow.cn/v1',
  API_KEY: process.env.TARO_APP_API_KEY,
  DEFAULT_MODEL: 'deepseek-ai/DeepSeek-V3.2',
  MAX_TOKENS: 4000,
  TEMPERATURE: 0.7,
  API_TIMEOUT: 60000,
  MAX_HISTORY_COUNT: 50,
  MIN_TASK_LENGTH: 5,
  MAX_TASK_LENGTH: 500,
};
```

> ⚠️ API Key 通过 Taro 环境变量 `TARO_APP_API_KEY` 注入，不硬编码在代码中。上线前迁移至后端代理。

### 4.3 本地存储设计

| 存储键 | 类型 | 说明 |
|--------|------|------|
| `sihua_history` | `HistoryRecord[]` | 历史记录，最多50条 |
| `sihua_settings` | `AppConfig` | 用户设置 |

### 4.4 开发环境配置

微信开发者工具需勾选以下选项：

```
右上角"详情" → 本地设置 → ☑️ 不校验合法域名、web-view、TLS版本及HTTPS证书
```

## 六、UI 设计规范（Claude 极简风格 · 优化版）

> 本节基于对 Claude 官网 (claude.ai) 设计语言的深度研究，从配色、排版、图标、容器、信息层级五个维度全面优化。

### 6.1 设计理念

Claude 的设计语言核心可以概括为三个词：**温暖、克制、呼吸感**。

| 原则 | 说明 | 与旧版差异 |
|------|------|-----------|
| **温暖而非冷感** | Terra Cotta 暖色作为品牌锚点，取代纯暖黑主色 | 旧版用 `#1C1917` 暖黑做按钮，缺乏品牌辨识度 |
| **克制而非堆砌** | 去除 emoji、彩色标签、可见边框，用字重和灰度说话 | 旧版用 ⚡💭📋 等 emoji 和彩色 tag，视觉噪音大 |
| **呼吸而非填满** | 容器间用留白分隔，而非白卡片叠在暖白底上 | 旧版"卡片汤"效果——白卡片 + 暖白底 = 层级模糊 |
| **渐进而非一次** | 信息折叠展示，用户按需展开 | 旧版四维分析全部平铺，信息密度过高 |
| **高对比排版** | 标题与正文的字号跳跃 3x+，字重 700 vs 400 | 旧版字号梯度 36→32→28→24，跳跃不够，层级扁平 |

### 6.2 配色系统

```
品牌色（Claude 核心 Terra Cotta）：
  --color-brand:        #D97757    品牌主色（Terra Cotta 暖陶色）
  --color-brand-light:  #F5E6DE    品牌浅底（用于 hover / 浅底标签）
  --color-brand-dark:   #B85A3A    品牌深色（用于 pressed 状态）

中性色（Claude 风格暖灰系）：
  --color-bg:           #FAFAF9    页面背景（暖白）
  --color-surface:      #FFFFFF    卡片 / 浮层背景
  --color-border:       rgba(0,0,0,0.06)   分割线（极淡，几乎不可见）
  --color-text-1:       #1C1917    主文字（暖黑）
  --color-text-2:       #78716C    次文字（比旧版 #57534E 更浅，降低对比噪音）
  --color-text-3:       #A8A29E    辅助文字
  --color-text-4:       #D6D3D1    占位文字

四化能量色（仅在需要色彩区分时使用，日常以灰度为主）：
  --color-lu:           #6B9E7A    禄（柔绿，比旧版更浅更柔）
  --color-quan:         #C49A5C    权（暖金，比旧版更亮更暖）
  --color-ke:           #7BA3C4    科（雾蓝，比旧版更浅更柔）
  --color-ji:           #C47A7A    忌（柔红，比旧版更浅更柔）

交互色：
  --color-primary:      #D97757    主按钮（品牌色，取代旧版暖黑）
  --color-primary-hover:#E8885F    主按钮 hover
  --color-primary-text: #FFFFFF    主按钮文字
```

**配色策略变化：**

| 场景 | 旧版 | 优化版 |
|------|------|--------|
| 主按钮 | 暖黑底白字 `#1C1917` | 品牌色底白字 `#D97757` |
| 四化标签 | 低饱和色块 + 白字图标 | 灰度文字 + 左侧 2rpx 色条 |
| 分割线 | 实色 `#E7E5E4` | 半透明 `rgba(0,0,0,0.06)` |
| 次文字 | `#57534E`（偏深） | `#78716C`（偏浅，减少视觉竞争） |

### 6.3 字体规范

Claude 官网使用 **serif 字体** 作为正文，营造学术可信感。小程序中我们用系统 serif 字体栈实现同样效果：

```
字体栈：
  正文：  ui-serif, Georgia, "PingFang SC", "Noto Serif SC", serif
  标题：  ui-sans-serif, -apple-system, "PingFang SC", sans-serif

标题（Display）：  font-family: sans-serif   letter-spacing: -0.03em
  · 页面大标题：  48rpx   font-weight: 700   line-height: 1.2
  · 区块标题：    32rpx   font-weight: 600   line-height: 1.3

正文（Body）：     font-family: serif
  · 主要内容：    28rpx   font-weight: 400   line-height: 1.75（比旧版 1.6 更宽松）
  · 辅助说明：    24rpx   font-weight: 400   line-height: 1.6
  · 标签/微标：   22rpx   font-weight: 500   letter-spacing: 0.02em

关键变化：
  · 标题用 sans-serif（粗壮有力），正文用 serif（温润可读）
  · 页面大标题从 40rpx → 48rpx，与正文形成 1.7x 跳跃
  · 正文行高从 1.6 → 1.75，增加呼吸感
  · 标题 letter-spacing 从 -0.02em → -0.03em，更紧凑有力
```

### 6.4 间距系统

```
基于 8rpx 网格（与旧版一致，但使用方式不同）：
  xs:  8rpx    组件内微间距
  sm:  16rpx   组件内间距
  md:  24rpx   组件间间距
  lg:  40rpx   区块间间距（旧版 32rpx → 40rpx，增加呼吸感）
  xl:  56rpx   页面级间距（旧版 48rpx → 56rpx）
  2xl: 80rpx   大区块间距（旧版 64rpx → 80rpx）

关键变化：
  · 区块间距整体放大 25%，消除"挤在一起"的感觉
  · 页面顶部留白从 48rpx → 64rpx，给标题更多呼吸空间
```

### 6.5 图标系统

**去除所有 emoji，改用线条图标。**

| 场景 | 旧版（emoji） | 优化版（线条图标） |
|------|-------------|-----------------|
| 简单模式 | ⚡ | `Zap` 图标（闪电线条） |
| 思考模式 | 💭 | `Brain` 图标（脑线条） |
| 历史记录 | 📋 | `Clock` 图标（时钟线条） |
| 我的 | 👤 | `User` 图标（人线条） |
| 最佳切入点 | 💡 | `Sparkles` 图标 |
| 整体建议 | 📌 | `Lightbulb` 图标 |
| 注意事项 | ⚠️ | `AlertTriangle` 图标 |
| 建议避免 | 🚫 | `XCircle` 图标 |
| 空状态 | 📭 | `Inbox` 图标（大号，线条感） |
| 跳过 | → | `ArrowRight` 图标 |
| 返回 | ← | `ArrowLeft` 图标 |

图标规范：
- 标准尺寸：32rpx × 32rpx
- 大号尺寸：48rpx × 48rpx（空状态等场景）
- 颜色：继承父元素文字颜色，不单独着色
- 描边宽度：1.5px（比默认 2px 更精致）

### 6.6 容器与卡片

**核心变化：从"白卡片叠暖白底"到"无感容器 + 留白分隔"。**

```
旧版模式：
  ┌─────────────────────┐
  │  白卡片（#FFFFFF）    │  ← 可见的矩形容器
  │  16rpx 圆角          │
  └─────────────────────┘
  （暖白底 #FAFAF9）

优化版模式：
  内容区块
  （无可见容器，靠留白和字重区分层级）

  仅在需要交互反馈时使用容器：
  · 输入区：极淡底色 #FAFAF9 + 无边框
  · 可点击项：hover 时 brand-light 底色
  · 折叠面板：左侧 2rpx 色条标识
```

| 元素 | 旧版 | 优化版 |
|------|------|--------|
| 卡片 | 白底 + 16rpx 圆角 + 可见边界 | 去除卡片壳，内容直接排列 |
| 输入区 | 暖白底 + 12rpx 圆角 | 暖白底 + 20rpx 圆角 + 无边框 |
| 按钮 | 16rpx 圆角 | 24rpx 圆角（更圆润，Claude 风格） |
| 分割线 | 实色 1rpx | 仅在必要时使用，0.5rpx + 半透明 |
| 四化维度块 | 灰底 + 彩色图标 + 白字 | 无底色 + 左侧色条 + serif 正文 |

### 6.7 微交互与动效

```
过渡时长：
  · 状态切换：200ms ease-out（与旧版一致）
  · 折叠展开：300ms cubic-bezier(0.4, 0, 0.2, 1)
  · 页面切换：250ms ease-in-out

新增动效：
  · 步骤勾选：checkbox 填充时 150ms scale(1.1) → scale(1) 弹性
  · 折叠展开：高度过渡 + 内容 fade-in
  · 按钮点击：100ms scale(0.97) 按压反馈
  · 加载状态：三个圆点依次淡入淡出（取代旧版四点旋转）
```

| 步骤标签 | 彩色 tag（高优·30分钟） | 纯文字（text-3 灰度），去除彩色 tag |
